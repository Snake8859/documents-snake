# WebGIS

首先，对以往WebGIS的观念进行修正，WebGIS不单单只是前端，而是一个综合性的技术体系或者理解为（前端+后端）地理信息式全栈，同时也是属于网络地理信息系统的一个重要组成成分。

  • 客户端部分

    若是做Web客户端，先要了解HTML，CSS，JS这些基础内容；然后再接触Web工程化框架，例如Vue，Angular；最后接触GIS框架，例如OpenLayer，Mapbox，Leaflet，ArcGIS API for JavaScript等。

    若是做移动端客户端，需要接触移动端开发基础；然后接触以安卓和IOS两大体系下的移动端框架。

  • 服务端部分

    服务端需要先了解一些Web应用服务器，例如IIS，Tomcat之类的；然后再接触计算机网络的知识；最后是GIS地理服务知识和工具，例如各类地图服务，ArcGIS Server和GeoServer等。

  • 其他

    若是想接触单纯前端的地图应用或者是位置应用（LBS），还可以多了解一些各大国内地图产商提供的LBS的API，比如高德地图，百度地图，腾讯位置服务等。

这一块的前置课程挺多的，先得学过HTML,CSS,JS这些，但是我觉得可以不用那么深入（如果不是特别想深入搞WebGIS的话😕），然后是一些Web应用服务器的了解，比如IIS，Tomcat之类的，再加上接触过网络接口使用，JS框架（JQuery，Dojo，Vue之类的），大致上了解过语法和基础环境搭建上，应该我觉得入个门问题不大。当时学校学这门课，应该是最头疼之一的了😖。

我觉得主要分两种吧，第一种以ESRI产品为代表，ArcGIS API for JavaScript和ArcGIS Server为核心的专业WebGIS开发。第二种以各大开源项目为代表，Leaflet，Mapbox GL JS和GeoServer为核心的开源WebGIS开发。当然，我觉得一些目前各大地图API应该也可以与之相互结合，比如高德地图API，百度地图开放平台，腾讯位置服务，也是值得使用的。
