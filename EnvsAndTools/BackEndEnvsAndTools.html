# Java环境搭建

目前Java由Oracle管理，可以到传送门下载对应版本的JDK。

具体环境配置的，可以参考传送门。

这里稍微关于安装的内容进行解释，在安装JDK的同时，会发现同时安装了JRE，而且在平时也会听到JVM，那么三者是什么以及其关系是什么呢？

  • JDK

    JDK即The Java Development Kit，Java开发工具包，为Java应用和applets开发提供基本开发环境，它包含Java Runtime Environment (JRE), an interpreter/loader (Java), a compiler (javac), an archiver (jar), a documentation generator (Javadoc) 以及其他Java开发所需要的工具。

    在安装完JDK后，在其目录下会发现发现有 /bin，/jre ，/lib，/include 文件夹，在网上调查后发现其主要的内容为

    • bin:各种工具的可执行文件(比如编译器javac.exe，执行器java.exe等)
    • include:java和JVM交互用的头文件
    • lib:类库
    • jre:java运行环境

  • JRE

    JRE即Java Runtime Environment，Java运行环境，有时候也被称为Java RET，即The Java Runtime Environment。它提供了运行一个Java应用的最基本环境，它包含JVM，核心类库等。

    在安装完JRE后，在其目录下也会发现有 /bin，/lib文件夹，在网上说可以认为bin里就是jvm，lib里是核心类库。

  • JVM

    JVM即Java Virtual Machine ，Java虚拟机，它是Java跨平台的关键，简单来说，它会将你写的Java程序（.java）编译为字节码文件(.class)，然后这个字节码文件在JVM上面运行，而不是直接交由操作系统，通过虚拟机与操作系统交互，以这样的方式达到跨平台的方式。举个简单的例子，在中国，你只要会说普通话就能与人正常交流，但是你想要再其他国家正常交流，难道就一定要会其他国家的语言吗，不一定，可以带一个这个国家的翻译家，通过你与翻译家沟通，翻译家再与当地人沟通，跨越这个语言障碍。那么JVM相当于这个翻译家的地位，你不需要直接当地人（操作系统）直接交互，通过中间件的方式即可实现跨语言（跨平台）。

简单来说，JDK=JRE+Development Tool，JRE=JVM+Library Classes

那么为什么有JDK包含了JRE和JVM，还需要装JRE和JVM呢，这个主要是在不同用户的角度来看就行。若你仅需要运行Java应用，那么需要给你JRE就满足了，若你是作为Java的开发者，你需要提供开发工具完成你的代码编译，运行等。以满足不同用户的最低需求。

# Linux常用命令

一般所使用的命令，可在该地方查询：传送门1或传送门2

常见命令：

  • 目录操作（增删改查）

    • 增

      • mkdir 目录名称 【创建文件夹】

        在根目录/ 下创建test文件夹 mkdir /test

    • 查

      • ls [-al] 【查看文件夹内容】

        ls -a 可以查看当前目录下所有文件和目录，包括隐藏文件

        ls -l 可以缩写成ll

      • find 目录 参数

        查询/root下的与test有关的目录（文件） find /root -name 'test*'

        更多命令，可在传送门中看到

    • 改

      • mv 目录名称 目录新的位置 【剪切】

        将newTest目录剪切到 /use下面 mv newTest /usr

        mv 语法不仅可以对目录进行剪切操作，对文件和压缩包都可以执行剪切操作

      • cp -r 目录名称 目录拷贝的目标位置 【拷贝目录】

        将/usr下的newTest拷贝到根目录下的test中 cp -r /usr/newTest /test

        -r 代表递归拷贝，cp命令不仅可以拷贝目录，还可以拷贝文件和压缩包，对文件和压缩包不需要使用-r

    • 删

      • rm [-rf] 目录【删除】

        删除/usr下的newTest，进入/usr 下使用rm -r newTest，在删除前会询问是否删除

        删除/test下的newTest，进入/test下使用 rm -rf newTest，不需要询问而强制删除

        rm不仅可以删除目录，也可以删除其他文件或者压缩包

  • 文件操作（增删改查）

    • 增

      • touch 文件名称

        在test目录下创建一个空文件 touch aaa.txt

    • 查

      • cat/more/less/tail 文件名

        cat/more/leess/tail aaa.txt

        用cat，只能显示最后一屏的内容

        用more，可以显示百分比，回车可以向下一行，空格可以向上一页，q可以退出

        用less，可以使用键盘上的PaUp和PaDn向上和向下翻页，q结束查看

        用tail -10 查看文件最后10行，Ctrl+C结束

        注意：命令tail -f 文件 可以对某个文件进行动态监控，例如动态查看tomcat的日志文件，可用tail -f 日志文件

    • 改

      • vim 文件

        编辑/test下的aaa.txt文件，使用vim aaa.txt。

        1. 但是此时不能编辑，因为还处于命令模式，点击键盘的i/a/o进入编辑模式
        2. 完成编辑后，按Esc，返回命令模式，输入冒号:，进行低行模型，输入wq!保存，输入q!强制退出不保存

        vim 文件------>进入文件----->命令模式------>按i进入编辑模式----->编辑文件 ------->按Esc进入底行模式----->输入:wq/q!

        vim更多内容，可参考传送门内看

    • 删

      • rm -rf 文件

  • 打包压缩文件操作

    Linux中的打包文件一般是以.tar结尾的，压缩命令一般是以.gz结尾的。而一般打包和压缩是一起进行的，打包并压缩后的文件的后缀名一般.tar.gz

    • 打包并压缩

      • tar -zcvf 打包压缩后的文件名 要打包的压缩文件

        其中

        • z：调用gzip压缩命令进行压缩
        • c：打包文件
        • v：显示运行过程
        • f：指定文件名

        打包/test下的所用文件，压缩打包的文件，指定名为成xxx.tar.gz

        tar -zcvf xxx.tar.gz aaa.txt bbb.txt cc.txt 或 tar -zcvf xxx.tar.gz /test/*

    • 解压压缩包

      • tar [-xvf] 压缩文件

        其中：

        • x：代表解压

        将/test下的xxx.tar.gz解压到当前目录 tar -xvf xxx.tar.gz

        将/test下的xxx.tar.gz解压到指定目录 tar -xvf xxx.tar.gz -C /usr

  • 其他命令

    • 显示当前位置 pwd

    • 搜索命令 grep

    • 管理命令 |

    • 查看进程 ps -ef

    • 杀死进出 kill -9 进程pid

    • 网络通信

      • 查看网卡信息 ifconfig
      • 查看与某台机器的连接情况 ping
      • 查看当前系统的端口使用：netstat -an

    • 后台执行命令

      • nohup command > myout.file 2>&1 &

        具体参考传送门

# nginx部署

Nginx是一款高性能的http 服务器/反向代理服务器及电子邮件（IMAP/POP3）代理服务器。由俄罗斯的程序设计师Igor Sysoev所开发，官方测试nginx能够支撑5万并发链接，并且cpu、内存等资源消耗却非常低，运行非常稳定。

安装步骤：

  1. 从官方网站，下载nginx的Linux安装包，推荐Stable Version

  2. 预安装依赖环境

    • gcc环境：yum install gcc-c++

    • PCRE环境：yum install -y pcre pcre-devel

      PCRE(Perl Compatible Regular Expressions)是一个Perl库，包括 perl 兼容的正则表达式库。nginx的http模块使用pcre来解析正则表达式，所以需要在linux上安装pcre库。

    • OpenSSL环境：yum install -y openssl openssl-devel

      OpenSSL 是一个强大的安全套接字层密码库，囊括主要的密码算法、常用的密钥和证书封装管理功能及SSL协议，并提供丰富的应用程序供测试或其它目的使用。

      nginx不仅支持http协议，还支持https（即在ssl协议上传输http），所以需要在linux安装openssl库。

  3. 解压，编译和安装

    • 上传下载的nginx源码包，然后解压

    • 使用configure命令，创建makeFIle

      这是教程的configure：

      ./configure \
--prefix=/home/snake/nginx \
--pid-path=/var/run/nginx/nginx.pid \
--lock-path=/var/lock/nginx.lock \
--error-log-path=/var/log/nginx/error.log \
--http-log-path=/var/log/nginx/access.log \
--with-http_gzip_static_module \
--http-client-body-temp-path=/var/temp/nginx/client \
--http-proxy-temp-path=/var/temp/nginx/proxy \
--http-fastcgi-temp-path=/var/temp/nginx/fastcgi \
--http-uwsgi-temp-path=/var/temp/nginx/uwsgi \
--http-scgi-temp-path=/var/temp/nginx/scgi\
--with-http_stub_status_module --with-http_ssl_module
注意：启动nginx之前，上边将临时文件目录指定为/var/temp/nginx，需要在/var下创建temp及nginx目录

      这是个人的configure：

      ./configure \
--prefix=/home/snake/nginx \
--http-client-body-temp-path=/home/snake/temp/nginx/client \
--http-proxy-temp-path=/home/snake/temp/nginx/proxy \
--http-fastcgi-temp-path=/home/snake/temp/nginx/fastcgi \
--http-uwsgi-temp-path=/home/snake/temp/nginx/uwsgi \
--http-scgi-temp-path=/home/snake/temp/nginx/scgi \
--with-http_ssl_module \
--with-http_stub_status_module

      注意，最后with插件一定要装，可用于之后的使用ssl证书，实现https

    • make 编译

    • make install 安装

    可参考传送门1，传送门2

  4. nginx的启动，重启和关闭命令

    进入nginx安装的sbin目录

    • 启动： ./nginx
    • 关闭：./nginx -s quit
    • 重启：./nginx -s reload

    可参考传送门3

# MySQL部署

MySQL部署比较复杂，网上内容也比较多，这里参考教程里的步骤，若以后网上找到详细准确的，再进行更新。

安装步骤：

  1. 查询系统是否自带MySQL，以CentOS为例

    • 输入 rpm -qa |grep mysql
    • 若出现了mysql的相关依赖，则可用使用rpm -e --nodesp mysql依赖卸载

  2. 安装MySQL依赖

    yum -y install libaio.so.1 libgcc_s.so.1 libstdc++.so.6

    yum update libstdc++-4.4.7-4.el6.x86_64

  3. 上传MySQL到Linux，主要是rpm的尾缀文件

  4. 解压到指定目录

  5. 安装

    • 安装服务器端：rpm -ivh MySQL-server-5.6.22-1.el6.i686.rpm
    • 安装客户端：rpm -ivh MySQL-client-5.6.22-1.el6.i686.rpm

    若该过程出现一些报错，可参考传送门

MySQL设置：

  1. 启动MySQL service mysql start

  2. 将mysql加到系统服务中并设置开机启动

    • 加到系统服务：chkconfig --add mysql
    • 自动启动：chkconfig mysql on

  3. 登录mysql

    • 一般mysql安装好之后，会生成一个随机密码，存储位置在 /root/.mysql_secret

  4. 开启mysql远程登录

    默认情况下mysql为安全起见，不支持远程登录mysql，所以需要设置开启 远程登录mysql的权限

    grant all privileges on . to 'root' @'%' identified by '远程登录密码';

    flush privileges;

  5. 开发对外访问短口 3306

    /sbin/iptables -I INPUT -p tcp --dport 3306 -j ACCEPT

    /etc/rc.d/init.d/iptables save ---将修改永久保存到防火墙中

# Redis部署

Redis安装过程也比较简单，主要是以下几步

  1. 官网下载Redis源码包
  2. 上传到Linux系统，解压
  3. 进入redis源码目录，用make命令编译
  4. 接着使用 make install PREFIX=安装目录，来安装

Redis设置：

  1. redis启动，在redis的安装目录的bin目录下启动./redis-server
  2. 设置后台启动
    • 把源码包中的redis.conf 复制到 安装目录bin 目录下
    • 使用配置文件启动 ./redis-server redis.conf
  3. 查看redis的进程 ps aux|grep redis
